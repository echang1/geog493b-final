1. Introduction¶

In this study, we compare the performance of two popular deep learning architectures, Convolutional Neural Networks (CNNs) and Vision Transformers (ViTs), on a scene classification problem. In colder regions of the world, drifiting icebergs in the ocean disrupt sea traffic due to the high maritime safety risks. We aim to develop a robust model for identifying whether a satellite image contains an iceberg or a ship in order to build an aerial-based environmental condition mointoring system.

2. Background¶

The CNN model has been widely used to solve image classification problems due its ability to automatically learn spatial relationships from raw pixel data. They have been used to achieve state-of-the-art performances on standard computer vision datasets including ImageNet, CIFAR-10, and MNIST [1,2,6], as well as on remote sensing-specific datasets [3,9]. However, the introduction of the transformer architecture, orignally designed to handle text data, has introduced a new method for image classification with the rapid development of vision transformer models. ViTs have been proven to perform at similar levels as CNNs while offering benefits such as robustness to both in and out-of-distribution data noise [4,8]. In the field of remote sensing, ViT models have shown to be comparable to and in some cases exceeeded the capabilities of other state-of-the-art classification models [5,7].

3.1 Loading Data into DataLoaders¶

We compare our classification models using the Statoil/C-CORE Iceberg dataset. This dataset contains radar backscatter images, hand labeled by human experts, which either contain an iceberg or a ship. We discarded the provided "test" split as it was unlabeled and split the "train" portion (1604 samples) with a 70%-10%-20% train-validation-test split (Samples: 1122, 161, 213). Since all images were fixed at a 75x75 resolution, no preprocessing was done on the raw images.

class IcebergDataset(Dataset):
    def __init__(self, data, labels=None):
        self.data = data
        self.labels = labels

    def __len__(self):
        return len(self.data)

    def __getitem__(self, idx):
        image = self.data[idx]
        if self.labels is not None:
            label = self.labels[idx]
            return image, label
        return image

# Load and preprocess the data
def load_data(file_path):
    with open(file_path, 'r') as f:
        data = json.load(f)
    
    images = []
    labels = []

    for item in data:
        band_1 = np.array(item['band_1']).reshape(75, 75)
        band_2 = np.array(item['band_2']).reshape(75, 75)
        image = np.stack([band_1, band_2], axis=0)
        images.append(image)
        labels.append(item['is_iceberg'])

    return np.array(images), np.array(labels) if labels else None

3.2 Model Architecture¶

CNN:
Our CNN contains four layers. The first two are convolutional and the other two are fully-connected. The fully-connected layer produces a 2-dimensional output, with the max value corresponing to the predicted class. Softmax was not used as this is a binary classification problem. The network minimizes cross-entropy loss using the Adam optimizer during the training process. Each of the convolutional blocks contain a convolutional layer with a 3x3 kernel, stride of 1, and padding of 1. ReLU activations were used, followed by max pooling layers with 2x2 kernels and strides of 2. The first block had 32 filters, while the second block used 64 filters, feeding an 18x18 feature map into the fully-connected layers. The output from the convolutional blocks was flatened and passed through two fully-connected layers, also using ReLU activation, before passing the 2 dimensional output. The model was trained for 100 epochs using the Adam optimizer with a learning rate of 1e-4 and batch size of 32. Validation loss was used as a performance metric to determine the best model without overfitting; all infernce was conducted using the model parameters with the lowest validation loss. All computations were performed on an NVIDIA RTX 4090 GPU using PyTorch. Data loading was handeled using the PyTorch Dataset and DataLoader functions.

class BasicCNN(nn.Module):
    def __init__(self):
        super(BasicCNN, self).__init__()
        # Convolutional blocks
        self.conv_layers = nn.Sequential(
            nn.Conv2d(2, 32, kernel_size=3, stride=1, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(kernel_size=2, stride=2),
            nn.Conv2d(32, 64, kernel_size=3, stride=1, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(kernel_size=2, stride=2)
        )
        # FC layers
        self.fc_layers = nn.Sequential(
            nn.Flatten(),
            nn.Linear(64 * 18 * 18, 128),
            nn.ReLU(),
            nn.Linear(128, 2)
        )

    def forward(self, x):
        x = self.conv_layers(x)
        x = self.fc_layers(x)
        return x

ViT:
Our ViT contains 3 main components: A convolutional layer serving as a patch embedding layer, A transformer encoding layer, and a fully connected layer serving as the classification output. The patch embedding layer divides input images into 16x16 patches, essentially "tokenizing" the input images into sequences of patch embeddings for the transformer encoder. The core component of the ViT is the multi-layer transformer encoder, consisting of 6 layers. Each layer contains a 8-head self attention mechanism as well as a 512-dimension feedforward neural network. The transformer output is then fed into a final fully-connected layer to produce a 2-dimensional output. Our model was trained for 100 epochs using cross-entropy loss and the Adam Optimizer with a learning rate of 1e-4 and batch size of 32. Validation accuracy was used as the performance metric to determing the best model without overfitting, as commonly done for generative neural networks compared to discriminative neural networks. All infernce was conducted using the model parameters with the lowest validation loss. All computations were performed on an NVIDIA RTX 4090 GPU using PyTorch. Data loading was handeled using the PyTorch Dataset and DataLoader functions.

class BasicViT(nn.Module):
    def __init__(self, image_size, patch_size, num_classes, dim, depth, heads, mlp_dim, channels=3):
        super(BasicViT, self).__init__()

        self.image_size = image_size
        self.patch_size = patch_size
        self.num_classes = num_classes
        self.dim = dim
        self.depth = depth
        self.heads = heads
        self.mlp_dim = mlp_dim
        self.channels = channels

        # Create patches
        self.patch_embedding = nn.Conv2d(self.channels, self.dim, kernel_size=self.patch_size, stride=self.patch_size)

        # Transformer encoder
        self.encoder_layer = nn.TransformerEncoderLayer(d_model=self.dim, nhead=self.heads, dim_feedforward=self.mlp_dim)
        self.encoder = nn.TransformerEncoder(self.encoder_layer, num_layers=self.depth)

        # Classifier head
        self.classifier = nn.Linear(self.dim, self.num_classes)

    def forward(self, x):
        # Convert the image into patches
        x = self.patch_embedding(x)  # (batch_size, dim, num_patches_height, num_patches_width)
        x = x.flatten(2)  # Flatten the height and width dimensions (batch_size, dim, num_patches)

        # Transform and add positional encoding
        x = x.permute(2, 0, 1)  # (num_patches, batch_size, dim)
        x = self.encoder(x)

        # Use the output of the transformer for classification
        x = x.mean(dim=0)  # Global average pooling (over patches)
        logits = self.classifier(x)
        
        return logits

def classification_accuracy(preds, y):
    # Convert preds to a tensor if it's a list
    preds = torch.tensor(preds)
    y = torch.tensor(y)
    
    # Compute accuracy
    correct = (preds == y).float()
    acc = correct.sum() / len(correct)
    return acc.item()

def train_basic_cnn(model, train_loader, val_loader, epochs=10, lr=1e-4, save_weights=False):
    criterion = nn.CrossEntropyLoss()
    optimizer = optim.Adam(model.parameters(), lr=lr)
    train_losses, train_accs, val_losses, val_accs = [], [], [], []

    if save_weights:
        output_dir = "./models/"
        if not os.path.exists(output_dir):
            os.makedirs(output_dir, exist_ok=True)

    best_val_loss = float('inf')
    best_val_acc = 0.0
    best_val_epoch = 0

    for epoch in range(epochs):
        model.train()
        running_loss = 0.0
        running_preds, running_targets = [], []
        for images, labels in train_loader:
            images, labels = images.to(device), labels.to(device)
            optimizer.zero_grad()
            outputs = model(images)
            loss = criterion(outputs, labels)
            loss.backward()
            optimizer.step()
            running_loss += loss.item()
            train_preds = torch.argmax(outputs, dim=1).cpu().numpy()
            running_preds.extend(train_preds)
            running_targets.extend(labels.cpu().numpy())

        running_accuracy = classification_accuracy(running_preds, running_targets)

        val_loss = 0.0
        val_preds, val_targets = [], []
        model.eval()
        with torch.no_grad():
            for images, labels in val_loader:
                images, labels = images.to(device), labels.to(device)
                outputs = model(images)
                loss = criterion(outputs, labels)
                val_loss += loss.item()
                preds = torch.argmax(outputs, dim=1).cpu().numpy()
                val_preds.extend(preds)
                val_targets.extend(labels.cpu().numpy())

        val_accuracy = classification_accuracy(val_preds, val_targets)
        train_losses.append(running_loss/len(train_loader))
        train_accs.append(running_accuracy/len(train_loader))
        val_losses.append(val_loss/len(val_loader))
        val_accs.append(val_accuracy)

        print('\n')
        print("="*120)
        print(f"Epoch: {epoch+1}/{epochs}" )
        print("="*120)
        print(f"Train Loss: {running_loss/len(train_loader):.4f}, Train Accuracy: {running_accuracy:.4f}\t|\tVal Loss: {val_loss/len(val_loader):.4f}, Val Accuracy: {val_accuracy:.4f}")
        if val_loss < best_val_loss:
            best_val_loss = val_loss
            best_val_acc = val_accuracy
            best_val_epoch = epoch+1
            if save_weights:
                torch.save(model.state_dict(), os.path.join(output_dir, f'best_basic_cnn.pth'))
            print(f"Model saved at epoch {epoch+1}")

    return train_losses, train_accs, val_losses, val_accs, best_val_acc, best_val_epoch

def train_basic_vit(model, train_loader, val_loader, epochs=10, lr=1e-4, save_weights=False):
    criterion = nn.CrossEntropyLoss()
    optimizer = optim.Adam(model.parameters(), lr=lr)
    train_losses, train_accs, val_accs = [], [], []

    if save_weights:
        output_dir = "./models/"
        if not os.path.exists(output_dir):
            os.makedirs(output_dir, exist_ok=True)

    best_val_accuracy = 0.0
    best_val_epoch = 0

    for epoch in range(epochs):
        model.train()
        running_loss = 0.0
        correct_preds = 0
        total_preds = 0

        for images, labels in train_loader:
            images, labels = images.to(device), labels.to(device)
            optimizer.zero_grad()
            outputs = model(images)
            loss = criterion(outputs, labels)
            loss.backward()
            optimizer.step()
            running_loss += loss.item()

            _, preds = torch.max(outputs, dim=1)
            correct_preds += torch.sum(preds == labels).item()
            total_preds += labels.size(0)

        epoch_loss = running_loss / len(train_loader)
        epoch_accuracy = correct_preds / total_preds * 100
        
        model.eval()
        all_preds = []
        all_labels = []

        with torch.no_grad():
            for images, labels in val_loader:
                images, labels = images.to(device), labels.to(device)

                # Forward pass
                outputs = model(images)
                _, predicted = torch.max(outputs, 1)

                all_preds.extend(predicted.cpu().numpy())
                all_labels.extend(labels.cpu().numpy())

        # Compute accuracy and classification report
        val_accuracy = classification_accuracy(all_labels, all_preds)
        train_losses.append(epoch_loss)
        train_accs.append(epoch_accuracy)
        val_accs.append(val_accuracy)

        print('\n')
        print("="*120)
        print(f"Epoch: {epoch+1}/{epochs}" )
        print("="*120)
        print(f"Train Loss: {running_loss/len(train_loader):.4f}, Train Accuracy: {epoch_accuracy:.4f}\t|\tVal Accuracy: {val_accuracy:.4f}")
        if val_accuracy > best_val_accuracy:
            best_val_accuracy = val_accuracy
            best_val_epoch = epoch+1
            if save_weights:
                torch.save(model.state_dict(), os.path.join(output_dir, f'best_basic_vit.pth'))
            print(f"Model saved at epoch {epoch+1}")

    return train_losses, train_accs, val_accs, best_val_accuracy, best_val_epoch

def evaluate(model, test_loader, criterion):
    model.eval()
    loss = 0.0
    all_preds = []
    all_labels = []
    
    with torch.no_grad():
        for images, labels in test_loader:
            images, labels = images.to(device), labels.to(device)
            outputs = model(images)
            loss += criterion(outputs, labels).item()
            _, predicted = torch.max(outputs, 1)
            all_preds.extend(predicted.cpu().numpy())
            all_labels.extend(labels.cpu().numpy())

    acc = classification_accuracy(all_labels, all_preds)
    return loss / len(test_loader), acc

4. Results and Discussion¶

_In this study, we compared the performance of Convolutional Neural Networks and Vision Transformers for scene classification on the Statoil/C-CORE Iceberg dataset. Our goal was to assess the effectiveness of each model in identifying whether an image represented an iceberg or a ship. The CNN model achieved a test accuracy of 85.36%, with the best performance observed at epoch 53, while the ViT model attained a test accuracy of 82.87% at epoch 44. As shown in Figure 1, the model's performance worsens after epoch 44 evident by the increasing trend in validation loss after this point, despite the consistently decreasing training loss.

The CNN model demonstrated strong performance with a test accuracy of 85.36%. This result highlights the CNN's ability to capture local patterns and spatial hierarchies within the radar backscatter images, which are key for distinguishing between icebergs and ships.

The ViT model, on the other hand, achieved a test accuracy of 82.87% at epoch 44. While this performance is still respectable, it falls short when compared to the CNN's results. Vision Transformers, which are based on self-attention mechanisms, excel at capturing long-range dependencies in the data. However, their performance can be sensitive to the size of the dataset and the quality of the feature representations. With the relatively small dataset size, the ViT may not have been able to fully capitalize on its capabilities. This is evident in figure 2, as the validation accuracy for the ViT seems to continue trending upwards long after the CNN validation accuracy trend has plateaued.

Both models reached their peak performances around 50 training epochs, which suggests that both architectures require similar computational resources. However, this been shown to not be the case in other work, with transformers requiring much more computation power in training. This may again be due to the limited dataset size and remains to be investigated in future work._

5. Conclusion¶

In this project, we explored the performance of two distinct deep learning architectures, Convolutional Neural Networks and Vision Transformers, for scene classification tasks using the Statoil/C-CORE Iceberg dataset. Our experiments focused on evaluating these models in identifying icebergs and ships in radar backscatter images. The results were promising, demonstrating the potential of both models in identifying icebergs and ships in radar backscatter images.

The Basic CNN model, with its convolutional layers, was able to capture local patterns effectively, showing competitive results in terms of accuracy. On the other hand, the Vision Transformer, utilizing self-attention mechanisms, performed well on the dataset, leveraging global context and capturing long-range dependencies across the image.

Further studies could expand on this research by testing these models on larger datasets with more varied and complex data. For example, including higher-resolution satellite imagery, multi-modal data (e.g., combining optical and radar imagery), or datasets with more classes in addition to the ships and icebergs. Additionally, experimenting with hybrid models combining CNNs for local feature extraction and ViTs for global context might offer a powerful approach for handling increasingly complex datasets.

Ultimately, the continued exploration of CNNs and ViTs on more complex datasets will contribute valuable insights into the strengths and limitations of these models for remote sensing and image classification tasks. By addressing these challenges, future research can refine deep learning techniques for more accurate and scalable solutions in real-world applications.
